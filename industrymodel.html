Energy Orchestration Model Spec v3

This page answers one concrete question (TVOL, BIW doorline): How can we reduce energy cost per door (€/door) — without risking quality, takt time, or OEE?
We do it with governed orchestration (CWD agent structure) on top of a stable Industrial Knowledge Layer (semantics + deterministic verifiers + optional learned accelerators).

1) The Core Problem

Objective

Minimize €/door within canonical 5-minute windows.

Metric = Energy Cost / Produced Units Resolution = 5m snapshot

We do not “optimize the factory”. We produce bounded, auditable recommendations that remain safe under industrial reality.

Non-Negotiables (hard constraints)

  • Quality gate OK (geometry / surface / gap&flush)
  • Scrap ≤ hard threshold
  • Rework ≤ hard threshold
  • Takt / cycle time OK
  • OEE OK
No trade-off policy Violation → HOLD
Fail-safe posture (normative)
Missing required measured input → HOLD (no recommendation).
Unclear semantics (tag identity, unit mismatch, time misalignment) → HOLD until normalized.
Constraints violated → HOLD. No silent execution.

2) Industrial Model Stack (Knowledge Layer)

Principle

In industry, “understanding” is not reading text. It is structure + constraints + context + lifecycle. Therefore, the stable core is a Knowledge Layer (semantics + deterministic verifiers). Agents orchestrate on top.

Stable representation Semantic alignment Deterministic truth Optional learned accelerators LLM ≠ industrial brain
Layered architecture (normative)
0 Data access → 1 Semantics → 2 Deterministic verifiers → 3 Optional learned models → 4 Agent orchestration (CWD) → 5 Governance (GEORGE/Guardian)

0) Data access (plumbing)

  • OT/IT streams (energy meters, PLC tags via OPC UA)
  • MES throughput & cycle
  • Quality results / genealogy
  • Maintenance events (optional)
Read-only No PLC writeback

1) Semantics (meaning & identity)

  • Asset identity (line/cell/stage)
  • Tag normalization (names, units, timestamps)
  • Windowing (canonical 5m snapshots)
  • Variant mix comparability
Ontology-lite Digital thread hooks

2) Deterministic verifiers (truth & checks)

  • Required inputs present?
  • Freshness bounds OK?
  • Hard constraints OK? (quality/takt/OEE)
  • Conflict checks (no unsafe overlaps)
Explains HOLD Machine-checkable

3) Optional learned models (accelerators)

  • Surrogates (fast approximations where solvers are too slow)
  • Regime / anomaly encoders (rare events)
  • Forecast helpers (informational, not decisive)
Only if it pays No bypassing verifiers
Normative rule
If semantics are not stable (unit mismatch, tag identity unclear, timestamps misaligned), the system must not recommend. It must degrade to HOLD.

4) Coordinator–Worker–Delegator (CWD)

Normative statement
CWD does not decide what is true. It structures computation and proposals. Final admissibility is decided by Governance (GEORGE/Guardian) against the Knowledge Layer verifiers.

Coordinator

  • Loads canonical snapshot
  • Checks semantic validity (units/time alignment)
  • Runs required-input & freshness verifiers
  • Triggers Workers with bounded tasks
Creates run_id Trace anchors

Worker Agents (energy levers)

  • Load Profile Worker (where is energy going?)
  • Metrology Scheduling Worker (shift deferrable jobs)
  • High-Draw Avoidance Worker (no unsafe overlaps)
  • Pause Sequencing Worker (restart order / ramp)
Bounded outputs No actuator calls

Delegator

  • Aggregates worker proposals
  • Checks conflicts & constraint risk
  • Builds recommendation bundle
  • Hands to Governance for admissibility verdict
Bundle = structured Impact + evidence

5) Concrete Energy Levers (TVOL)

Metrology load shifting (Stage 6)

Move deferrable inspection jobs inside the window when (a) grid cost is lower or (b) peak demand overlap is avoided.

Within bounded delay Quality guardrail stays hard

Avoid simultaneous high draw

No overlap of high-draw operations (e.g., hemming peaks + high-energy inspection peaks).

Conflict verifier Risk → HOLD

Pause sequencing

Controlled restart logic after breaks (ramp-up order, avoiding simultaneous ramp peaks).

Bounded playbook Ticket-based adoption

Variant mix normalization

Ensure €/door comparisons are structurally valid: same variant band or normalized baseline.

Semantic layer Unclear mix → HOLD
Hard boundary
Recommendations must never require “guessing” missing production truth. If energy/throughput/quality/cycle/OEE evidence is missing or stale → HOLD.

8) Reality Check (Why this holds up in practice)

Three problems we do NOT mix

  • Data access (plumbing)
  • Semantic alignment (meaning + identity)
  • Representation learning (optional ML accelerators)
Semantics before ML No “train on chaos”

Pragmatic decision rules

  • If tags/units/time are inconsistent → fix semantics; do not “IFM” it.
  • If verification is missing → add deterministic verifiers first.
  • If a solver is too slow → use a surrogate (optional) but always verify.
  • If benefit is unclear → don’t add ML. Keep it governed and simple.
Agents on structured stack win
Bottom line
We industrialize energy optimization by making it admissible: stable semantics + deterministic truth + orchestrated proposals + governance/veto + audit.

9) What This Is Not

  • No PLC write access
  • No autonomous physical execution
  • No ERP/MES replacement
  • No blackbox AI that “just decides”
Non-claim (explicit)
This spec does not claim model “understanding” via a single monolithic foundation model. It claims an architecture that remains reliable under industrial reality: messy data, drift, and governance constraints.
