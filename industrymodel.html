Condensed World Model

A single representation of what exists (entities), what is true right now (state), what is allowed (constraints & policies), what is remembered (memory), what can be tested safely (simulation), and what can be executed (tools & process flow). For real factories, that only becomes credible when it is paired with audit-grade evidence.

Doorline as the First Real Instance

Context

Doorline is a manufacturing line within the Bodyshop subsystem. For the showcase, we zoom into one concrete product path: BIW Door Front Left (TVL).

Why it matters

A world model becomes operational only when it is mapped to real entities and real signals — and when decisions can be audited with evidence (not “explained” after the fact).

Outcome

Doorline becomes the anchor for consistent state, controlled actions, traceable decisions, and resilient operation — inside one governed model.

One-sentence showcase

A BIW doorline turns stamped parts into a geometry- and surface-critical product — across pre-assembly, reinforcement, hemming, structural completion, and a hard quality gate for gap/flush and surface.

What the model contains

These building blocks define how the factory is represented and how it moves from sensing to governed execution — with an audit-first mindset: every meaningful change must produce evidence.

Entities

Real things in the factory model (systems, stations, fixtures, products).

State

The current truth: conditions, availability, readiness, signals, and quality status.

Constraints

Hard limits that must never be violated (technical and operational).

Policies

Rules that shape permissible actions, approvals, and escalation paths.

Memory

What the system retains to remain consistent and improve over time (governed).

Simulation

Safe evaluation of possible decisions before committing to execution.

Tools

Controlled capabilities to observe, validate, and (only when allowed) act.

Process Flow

The ordered path from sensing to governed execution, including recovery.

Intent

Desired outcomes expressed as goals (e.g., “gap/flush within spec”), not manual steps.

Governance

Guardrails, approvals, accountability, and evidence for decisions.

Doorline (TVL) — Process Stages 1–6

The following stages describe a realistic BIW flow for a Front Left Door. Each stage is written in “world-model form”: entities, signals, state updates, decisions, and audit evidence.

Starting point (Input)

Stamped/formed parts arrive from press shop / kitting: Inner Panel, Outer Panel, reinforcements (side impact beam, hinge/lock reinforcements, beltline/frame areas), plus optional inserts (studs, nuts, brackets, guides). The target state is a geometrically stable, joined BIW door ready for downstream (paint, assembly).

Part ID (barcode/QR/RFID) Variant / sequence (TVL) Kitting completeness Door ID created
Process Stage 1 Pre-Assembly Inner Structure

Inner build-up (start the base body)

The door “begins” as an inner structure: the inner panel is positioned in fixtures and first reinforcements are placed and tack-fixed.

Entities
  • Inner Panel, fixture/clamps, base reinforcements, Door ID (TVL)
Signals (explicit sensors / scanners)
Barcode/QR/RFID reader 2D/3D vision (pose check) Clamp position sensors
State update
  • door.variant confirmed
  • fixture.clamped = true
  • parts.present = inner + initial reinforcements
Decision & governance
  • Edge validates “part present + orientation OK” → otherwise HOLD
  • Orchestrator schedules transfer only if constraints pass
  • Guardian blocks continuation if variant mismatch or clamp not OK
Evidence
  • Pose snapshot + clamp confirmation stored under Door ID
  • Decision record: OK / HOLD with rule reference
Process Stage 2 Reinforcements Functional Zones

Hinge, lock & crash structure

Critical functional zones are reinforced (hinge, lock, side impact). Presence and coarse positioning are verified before transfer.

Entities
  • Hinge/lock reinforcements, crash beam, studs/nuts/brackets, joining points
Signals (explicit sensors / scanners)
Structured-light 3D scan Vision presence check Force/torque sensors (joining)
State update
  • reinforcements.hinge/reinforcements.lock/reinforcements.crash = present
  • join.tack_complete = true
Decision & governance
  • Edge flags missing reinforcement → HOLD or rework route
  • Orchestrator ensures sequence correctness and station readiness
  • Guardian enforces “no transfer without presence evidence”
Evidence
  • 3D alignment summary (pass/fail + delta)
  • Joining force/torque trace attached to Door ID
Process Stage 3 Outer Panel Bonding & Hemming

Outer panel joining (surface risk begins)

The outer panel is applied and hemmed. This stage heavily influences surface appearance and later gap/flush behavior.

Entities
  • Outer panel, adhesive/sealer, hemming tooling (pre-hem / final hem)
Signals (explicit sensors / scanners)
Vision/line scan (surface defects) 3D scan (alignment & flush) Sealer thickness / bead monitoring
State update
  • outer_panel.aligned = true/false
  • hemming.pre and hemming.final results stored
Decision & governance
  • Edge detects surface risk (dent/scratch imprint) → HOLD / rework
  • Orchestrator may run a “proceed risk” simulation step (policy-gated)
  • Guardian enforces “no final hem without alignment evidence”
Evidence
  • Surface map snapshot + defect flags (if any)
  • Adhesive/hemming parameters appended to trace
Process Stage 4 Structural Completion Finalize Joining

Finalize structure & completeness

Remaining joining operations are completed and verified. The objective is structural completeness and stability.

Entities
  • Final weld/rivet/clinch points, hard-to-reach joins, BIW features
Signals (explicit sensors / scanners)
Weld process monitoring (current/time) Force/torque (rivet/clinch) 3D spot verification scan
State update
  • join.complete = true
  • integrity.score updated (rule-based)
Decision & governance
  • Edge flags missing join evidence → HOLD
  • Orchestrator routes to rework if allowed by policy
  • Guardian prevents “OK” if any mandatory join lacks evidence
Evidence
  • Join parameter logs + completeness report linked to Door ID
Process Stage 5 Hard Quality Gate Geometry + Surface

Quality Gate: geometry & surface

The door is released or diverted. This is the “audit-grade” gate: gap/flush and surface drive downstream cost and customer perception.

Entities
  • Measurement plan, tolerance rules, door surfaces, datum points, audit case
Signals (explicit sensors / scanners)
High-precision 3D scanner Surface vision / defect scan Gap/flush measurement
State update
  • quality.geometry = pass/fail + deltas
  • quality.surface = pass/fail + defect tags
  • classification = OK / REWORK / SCRAP / HOLD
Decision & governance
  • Edge performs deterministic pre-checks (no “intuition”)
  • Orchestrator validates rule application and creates an audit case if needed
  • Guardian enforces release logic and evidence completeness
Evidence
  • Scan artifacts (IDs + summaries) attached to Door ID
  • Rule references + classification decision stored as audit trace
Process Stage 6 Final Audit Handover

Final audit & handover

The inspected door is formally handed over — physically and digitally — with a final state and a complete evidence chain.

Entities
  • Door ID, batch/sequence, logistics buffer, downstream consumer, evidence links
Signals (explicit sensors / scanners)
Label print & scan verification RFID/QR tracking scan System status handshake
State update
  • door.state = RELEASED / QUARANTINE / HOLD
  • handover.complete = true
Decision & governance
  • Orchestrator issues final handover only if Stage 5 evidence exists
  • Guardian blocks shipment on missing audit references
Evidence
  • Final trace entry links all prior stage evidence + final disposition
Audit-first interpretation

This process description is not “documentation for humans only”. It is intentionally structured so it can be executed and audited: each stage updates state, produces evidence, and remains governance-controlled.

Identification

Barcode / QR RFID

Ensures parts are present and correctly seated before joining operations proceed.

2D / 3D vision Proximity sensors Clamp sensors

Joining process

Captures process parameters as evidence for structural integrity and completeness.

Weld monitoring Force / torque

Geometry

Structured-light 3D Laser scanning Gap/flush measurement

Surface

Detects dents, scratches, imprints — the key risk during outer panel joining & hemming.

Surface vision Line scan

Traceability

Guarantees every critical decision is traceable and linked to a Door ID.

From perception to action

The model follows a deliberate sequence — the factory thinks first, then acts.

  1. Perception — status and signals are captured.
  2. World Model Update — states are updated consistently.
  3. Reasoning & Simulation — possible decisions are evaluated.
  4. Governance Check — rules and approvals are enforced.
  5. Execution — only then actions are performed.
  6. Resilience — detect deviations, stabilize, and recover safely (with traceable outcomes).
